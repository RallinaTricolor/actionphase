# Phases

Phases are the units of gameplay within an **In Progress** game. The GM creates and manages phases; one phase is active at a time.

## Phase Types

- **Common Room** — Players read GM posts and comment. Private messages can be sent. Polls can be run.
- **Action Phase** — Players submit private actions. The GM writes and publishes results.

## Creating a Phase

Click **New Phase** in the Phase Management panel (Phases tab, GM only). The form includes:

- **Phase Type** — Common Room or Action Phase (required)
- **Title** — Optional custom name (e.g., "The Council Meets")
- **Description** — Optional context shown at the top of the phase (max 500 characters, Markdown supported)
- **Auto-activate at** — Optional date/time to activate the phase automatically. Leave blank to activate manually.
- **Deadline** — Optional deadline shown as a countdown to all participants

New phases are created in **Draft** status and are not visible to players until activated.

## Activating a Phase

Click **Activate** on a draft phase to make it the current active phase. Only one phase can be active at a time. If there are unpublished action results from the previous phase, you will be prompted to publish them before activating.

## Editing a Phase

Click **Edit** on any phase to update its title, description, auto-activate time, or deadline.

## Deleting a Phase

A phase can be deleted only if it has no associated content (submissions, results, messages, polls, or threads). If content exists, the phase cannot be deleted.
